Protokolle und Ports

Der Internet-Browser schickt Anfragen über HTTP, das
Hypertext-Transfer Protocol oder vorzüglich über das verschlüsselte HTTPS (= HTTP Secure). Ports sind wie Datenpacket-Tunnel. Betriebssysteme stellen Portanfragen um Verbindungen annehmen zu können. Es gibt Standard-Ports, für bestimmte Programme fest-zugeordnete Portnummern. HTTP läuft auf Port 80 über TCP. Das HTTP-S dagegen hat Port 443.

Wenn man im Windows Taskmanager (oder in Linux mit den Befehlen: top; ps; tree). die "PID" (Prozess ID) eines laufenden Programms findet kann man diesen verwenden um über den (CMD-)Terminal-Befehl netstat -nano, vorhanden in Windows und Linux, die Ports und deren Adressierung, sprich deren IPs, ausfinding machen.

Die Ports benutzen 2 Arten von Protokolle. Entweder TCP oder UDP.
Ein Programm kann über mehrere Ports laufen und beide Protokolle verwenden.
TCP = Transmission Control Protocol / UDP = User Datagramm Protocol

Über TCP verlaufen Daten "zuverlässig" im bidirektionalem Datenaustausch, ein Verlust von Datenpacketen wird hier verhindert. Der Schwerpunkt dieses Protokolls ist die verbindungsorientierte Paketvermittlung, wärend bei UDP nicht wichtig ist dass ALLES ankommt, sondern KONSTANT übertragen wird, mit geduldeden Unterbrechungen, beispielsweise beim Stream gucken oder bei Sprachübertragungen. Daher ist der Durchlass hier größer, dafür aber unzuverlässiger als über TCP.

FTP File Transfer Protocol
Transportprotokoll: TCP
Port 21 (Anfrage vom Client)
Port 20 (Übertragung)
Sichere Alternative: SFTP, Port 22

Telnet
Transportprotokoll: TCP
Port 23
Remote zugriff auf entfernte Computer
Datenbankzugriffe
Sichere Alternative: SSH, Port 22

SMTP Simple Mail Transfer Protocol
Transportprotokoll: TCP
Port 25
E-Mails in Computernetzwerken verschicken
IMAP und POP3 E-Mails in Computernetzwerken verschicken
Sichere Alternative: SMTP mit SSL/TLS, Port 465/587

DNS Domain Name System
Transportprotokoll: TCP/UDP
Port 53
Namensauflösung, wie “Telefon-Auskunft”
Sichere Alternative: AlternDNS mit SSL/TLS, Port 853ativen

DHCP Dynamic Host Configuration Protocol
Transportprotokoll: UDP
Server: Port 67
Client: Port 68
Zuweisung von IP-Adressen an Endgeräte
Vergabe einer eindeutigen IP-Adresse
Zuweisung des zuständigen Default-Gateways
Zuweisung einer Subnetzmaske
Zuweisung des zuständigen DNS-Servers

HTTP Hypertext Transfer Protocol
Transportprotokoll: TCP
Port 80
Zustandloses Protokoll
Methoden: GET, POST, HEAD, PUT, PATCH, DELETE, TRACE, OPTIONS, CONNECT
Statuscodes: 200 OK, 500 Internal Server Error, 404 Page Not Found, 451 Unavailable for legal reasons, 408 Request Timeout,...
Sichere Alternative: HTTPS, Port 443 (Neuer Standard)

POP3 Post Office Protocol 3
Transportprotokoll: TCP
Port 110
Client holt E-Mails vom E-mail-Server ab
Versand über SMTP
Sichere Alternative: POP3 mit SSL/TLS, Port 995

NTP Network Time Protocol
Transportprotokoll: UDP
Port 123
Synchronisation von Uhrzeiten | Bsp. SIEM (Splunk) Events beim Logging von mehreren Servern mit chronologisch protokolliert und geordneten Zeitstempel zur korrekten Durchführung von Analysen
Sichere Alternative: NTS (Networktime Security Protocol), Port 4460 (TCP statt UDP)

IMAP Internet Message Access Protocol
Transportprotokoll: TCP
Port 143
Bereitstellung von E-Mails auf Servern, die dann auf synchronisierten Geräten mit SMTP angefordert werden können
Mehr Funktionen als POP3
Sichere Alternative: IMAP mit SSL/TLS, Port 993

LDAP Lightweight Directory Access Protocol
Transportprotokoll: TCP/UDP
Port 389
Abfrage und Änderung (von Informationen) in verteilten Verzeichnisdiensten Speicherung von X.500 Objekten (Beschreibt den Aufbau von Verzeichnis-Diensten)
Authentifizierung, Authorisierung, Adress- und Benutzerverzeichnisse
Sichere Alternative: LDAP mit SSL/TLS, Port 636
