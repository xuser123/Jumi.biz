HTML „Hypertext Markup Language“
CSS „Cascading Style Sheets“

Grundlagen:
(HTML = Auszeichnungs-Sprache / Strukturierung)
(CSS = "Stylesheet"-Sprache / Gestaltungsvorgaben)

Jeder Quellcode einer Website beginnt mit <!doctype html>

Ein sogenannter „Tag“ mit Dokumenten-Typ-Definition.
Damit weiß der Browser dass es sich um ein HTML-Dokument handelt.
Üblicherweise folgt dann der html-Starttag <html>

oder mit Sprachenangabe. ( "en" = English / "de" = Deutsch /...)

<html lang="en">.

Dieser wird ganz am Ende des gesamten Seiten-Quellcodes mit dem Endtag </html>

abgeschlossen. Endtags erkennt man am Schrägstrich. Nach dem html-tag kommt der head-Tag, haupsächlich zum betiteln des Browserfensters.
Weitere Informationen wie Zeichenkodierung (UTF-8 für üäö) und Metadaten zur Website sind auch möglich. („Standalone-Tag“)
<meta charset="utf-8">

Suchmaschienen registrieren den Inhalt des "describtion"-Containers.

<meta name="description" content="IT methods for beginners in Project Website of Jumi">


Über das Hypertext-Transfer-Protokoll (HTTP) kommuniziert der Server zum Browser.



Der eigentliche Webseiteninhalt steckt im <body>-Bereich.
Im Beispiel mit einer Überschrift und Text darunter:


  <body>
  <h1>Große Überschrift</h1>
  <p>Paragraph für die Überschrift</p>

Mit dem "Script"-Tag werden Skripte (außerhalb HTML) eingebettet.

<script src="EinSkript.js"></script>

Dabei muss aber die Position dieses Tags beachtet werden, da im HTML-Dokument alles von oben nach unten ausgeführt wird Das Erscheinungsbild wird dann aus einer "CSS"-Datei ausgelesen und im Head-Bereich verlinkt. So wird vermieden dass man den Stil mehrerer Websites nicht wiederholt coden muss. Hier Zwei Varianten:

<link rel="stylesheet" href="style.css">
<link rel="stylesheet" type="text/css" href="cssdatei.css">

Der Code in der CSS-Datei hat eine eigene Scriptsprache die sich um die Darstellung der Inhalte im html-body kümmert. Hier hier steht das "p" für alle Paragraphen:


  p {
    font-family: arial, sans-serif;
    line-height: 150%;
    usw…
  }


Das geht mit allen HTML Haupt-Elementen, wie "body" und "div". "div"-Tags spielen eine große Rolle. Sie definieren so genannte "Blöcke". Diesen Blöcken kann man ID's geben. z.B.:

<div class="Objekt-Klasse" id="eineID">

Auf diese ID's wird dann in externen Skripten zurückgegriffen. Die Tag-Namen, "div's" & Objekte mit class oder id sind wie Code-Brücken zu anderen Scriptsprachen, welche die Seitenausgabe mitbestimmen.
